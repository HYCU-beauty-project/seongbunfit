# 성분핏 (IngredientFit)

Figma 시안 + 기획서(성분핏.pdf) 기반 프로토타입입니다. 기존에 잡아둔 파일 구조(뼈대) 위에
실제 동작하는 코드를 채워 넣었습니다.

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 http://localhost:3000 접속

> Next.js 16 기준입니다 (`AGENTS.md` 참고 — Turbopack이 기본이라 `--turbopack` 플래그가 필요 없어요).

## 페이지 구성

- `/` — 랜딩페이지. 히어로는 실사 이미지(`public/images/`), 가성비 산출 방식은 도넛 차트,
  "데이터로 비교하세요" 섹션은 좌측 텍스트+우측 성분핏/타사 패널로 Figma 배치에 맞췄어요
- `/chat` — 면책 동의 모달(팝업, 페이지 이동 없음) → AI 채팅 추천 플로우. 우측 하단에 문의하기 버튼
- `/products` — 채팅 없이 DB 제품 목록을 검색·필터(고민/성분)·정렬(가격/ml당가격/용량)로 열람.
  12개씩(3열×4행) 페이지네이션되고, 카드 클릭 시 큰 이미지의 상세정보 모달이 떠요. 없는 제품은
  "추가 요청하기" 버튼으로 실제 입력폼 모달(제품명/브랜드/링크/요청사항)을 거쳐 메일로 전송돼요.
  헤더의 "AI 추천받기" 버튼으로 언제든 채팅으로 이동 가능
- `/faq`(문의하기 버튼 포함), `/notice`(아코디언), `/terms` — 모두 하단에 공용 `Footer`(주소·이메일 포함) 표시
- `/mobile`, `/mobile/chat`, `/mobile/products`, `/mobile/faq`, `/mobile/notice`, `/mobile/terms`,
  `/mobile/programs` — 모바일 전용 페이지. 상단 고정 헤더의 햄버거 버튼을 누르면 우측에서
  메뉴가 슬라이드로 열려요(하단 탭바 대신 이 방식으로 통일했어요). 색상·컴포넌트는 전부 웹과
  같은 디자인 토큰을 써서, 별도 CSS 파일 없이 웹과 톤이 통일돼요. 화장품 검색/FAQ/공지사항/
  이용약관은 각각 `ProductsContent`/`FaqContent`/`NoticeContent`/`TermsContent` 공용 컴포넌트를
  데스크톱과 그대로 공유해요(내용 수정은 한 곳에서만 하면 양쪽에 다 반영돼요). 얼굴인식 피부
  진단은 이번 범위에서 제외했고, `/mobile/programs`에 "곧 만나요" 상태로만 자리를 잡아뒀어요.

## 채팅 플로우 (4단계)

1. **고민 입력**: 카테고리 칩 또는 자유 텍스트 → `POST /api/chat`이 카테고리 분석
   (`GEMINI_API_KEY` 있으면 Google Gemini, 없으면 키워드 매칭 폴백)
   - 피부 고민과 무관한 문장("안녕", "오늘 날씨 어때" 등)은 억지로 카테고리를 고르지 않고
     "그 부분은 도와드리기 어려워요"라고 답하며 카테고리 칩을 다시 보여줘요. (예전엔 이런
     경우에도 `categories[0]`으로 기본값 처리돼서 "안녕"에도 "주름 고민이시군요!"라고
     틀리게 답하는 버그가 있었는데 고쳤어요.)
2. **성분 추천 및 선택**: 카테고리별 핵심 성분 3개, 우측 패널에 효능/주의사항/적합 피부
3. **예산 설정**: 칩 선택 또는 자유 입력("2만원대" 자동 파싱)
4. **TOP3 결과**: `POST /api/recommend`가 가성비 점수 계산 후 TOP3 반환

대화 내역·선택 조건은 localStorage에 저장됩니다 (`lib/useLocalStorage.ts`,
`useSyncExternalStore` 기반이라 이 레포의 `react-hooks` lint 규칙과도 충돌 없어요).

**모바일 채팅은 UI가 달라요** (`ChatWindow`의 `forceStacked` prop): 우측 패널이 없고 전부
아래로 쌓여요. 화면이 좁아서 데스크톱과 똑같이 만들면 오히려 못 쓰게 되는 부분들을 모바일
전용 패턴으로 바꿨어요.
- **성분 카드**: 호버가 없는 터치 환경이라, 카드를 탭하면 정보가 펼쳐지고
  "이 성분으로 선택하기" 버튼을 따로 눌러야 확정돼요(`IngredientCard`의 `compact` 모드).
  기존 "지금 보는 성분" 우측 패널은 모바일에서는 안 보여요.
- **선택한 조건**: 한 줄 요약 바로 접혀있다가 탭하면 펼쳐져요(`CompactConditionsBar`).
- **가성비 계산법 보기 / 비교함**: 상시 노출 대신 우측 하단 동그란 버튼으로(`MobileActionFabs`).
- **TOP3 결과**: 좁은 화면에 카드 2~3개를 욱여넣는 대신, 한 번에 하나씩 스와이프/화살표로
  넘겨보는 캐러셀로(`ResultCarousel`).

**중간에 다시 선택하기**: 우측 "선택한 조건" 패널의 고민/성분/예산 태그를 클릭하면
전체를 재시작하지 않고 그 항목만 다시 고를 수 있어요. 완전히 처음부터 하고 싶으면
"처음부터 다시 시작하기" 버튼을 쓰면 돼요.

**계산법 투명하게 보여주기**: 채팅창 옆 "🧮 가성비 계산법 보기" 버튼을 누르면 공식 설명과
함께, 지금 받은 TOP3 결과가 있다면 제품별 실제 계산값(배치/가격/예산 점수 → 최종 점수)을
보여줘요. 각 점수는 숫자만 나열하지 않고 `ScoreBar`(공용 애니메이션 막대그래프 컴포넌트,
`components/ScoreBar.tsx`)로 0에서 실제 점수까지 차오르게 표시해서 한눈에 비교되도록 했어요.
맨 아래에는 전체 성분의 기준위치 목록도 있어요 — 성분마다 기준위치가 달라서 같은 순번이라도
점수가 다르게 나오는 이유를 확인할 수 있어요.

**결과 카드 이미지로 저장/공유**: 각 추천 카드 우측 상단의 ⤓ 버튼을 누르면 `html-to-image`로
카드를 PNG로 캡처해요. 모바일처럼 Web Share API를 지원하는 환경에서는 공유 시트가 바로 뜨고,
아니면 다운로드돼요.

**비교함**: 카드의 + 버튼으로 최대 4개까지 담아서 나란히 비교할 수 있어요. 우측 컬럼에
"비교함 N"으로 항상 보이게 뒀고(마우스 올리면 설명 툴팁도 떠요), 카드의 버튼들도 아이콘만이
아니라 "비교함에 담기" / "이미지 저장"처럼 글자로 표시해서 뭘 하는 버튼인지 바로 알 수 있게
했어요. 다른 고민(예: 주름 → 미백)으로 재검색해도 비교함은 유지돼서 서로 다른 카테고리
제품끼리도 비교 가능해요. 배치/가격/예산/가성비 지수 네 항목도 숫자 나열 대신 `ScoreBar`로
보여줘서, 카드 여러 개를 나란히 펼쳐놓고도 어느 제품이 어느 항목에서 우세한지 막대 길이로
바로 비교돼요.

**즐겨찾기**: 추천 카드 우측 상단의 ☆ 버튼을 누르면 저장되고(★로 바뀌어요), 다시 누르면
해제돼요. 비교함과 마찬가지로 로그인 없이 `localStorage`(`ingredientfit:favorites`)에
저장되는 방식이라 별도 회원 시스템 없이 바로 동작해요. 우측 유틸리티 카드의 "⭐ 즐겨찾기"
버튼(모바일은 우측 하단 플로팅 메뉴)을 누르면 지금까지 저장한 제품을 최신순으로 모아보는
`FavoritesModal`이 열리고, 여기서도 배치/가격/가성비 점수를 `ScoreBar`로 확인하고 바로
구매 링크로 이동할 수 있어요. 다만 로그인 기반이 아니라서 기기를 바꾸거나 브라우저 데이터를
지우면 목록이 사라져요 — 기기 간 동기화가 필요하면 나중에 로그인 기능과 함께 서버 저장으로
옮기면 돼요.

## 가성비 점수 계산

```
최종 점수 = (전성분 배치 점수 × 0.60) + (ml당 가격 점수 × 0.30) + (예산 점수 × 0.10)
```

자세한 산식은 `lib/scoring/calculator.ts` 상단 주석 참고.
(초기 랜딩페이지 시안엔 50/30/20이 적혀 있었는데, 확인 결과 60/30/10으로 최종 확정되었습니다.)

**전성분 순번에 베이스 성분 포함 여부**: 정제수·글리세린 같은 베이스 성분을 제외하지 않고,
원본 전성분 리스트 그대로 순번을 매기는 것으로 확정되었습니다(기준위치 숫자들은 이미 이를
감안해 정해진 값이에요). `lib/products.ts`의 더미 `actualPosition` 값 자체는 프로토타입용
예시 숫자라 실제 제품 데이터 연동 시 다시 채워질 값입니다 — 계산 공식은 이미 이 방식대로
동작하고 있었고, 화면 설명 문구만 반대로 적혀 있던 걸 바로잡았습니다.

## AI(Google Gemini) 실제로 연동하는 방법

1. https://aistudio.google.com/apikey 에서 API 키 발급 (신용카드 등록 불필요, 무료 티어 있음)
2. 프로젝트 루트의 `.env.local` 파일에 추가:
   ```
   GEMINI_API_KEY=AIza...
   ```
3. `npm run dev` 재시작 — 끝이에요. 코드 수정은 필요 없어요.

키가 있으면 `lib/gemini/analyzer.ts`가 `/api/chat` 요청마다 Gemini(`gemini-3.1-flash-lite`)를 호출해서
사용자 문장을 5개 카테고리 중 하나로 분류해요. 실패하거나 키가 없으면 자동으로 키워드 매칭으로
조용히 폴백돼요(에러가 나서 화면이 깨지는 일은 없어요).

무료 티어는 분당 요청 수 제한이 있어요 — 정확한 숫자는 Google이 모델/계정별로 자주 바꿔서,
필요하면 [Google AI Studio](https://aistudio.google.com)에서 프로젝트별 실시간 한도를 확인하세요.
데모/개발 용도로는 충분하지만, 짧은 시간에 너무 여러 번 연속으로 테스트하면 일시적으로
폴백될 수 있어요.

> ⚠️ **모델이 갑자기 404가 날 수 있어요.** Google이 예고 없이 특정 모델을 조기 중단하는 일이
> 실제로 있었어요(`gemini-2.5-flash`가 공식 종료 예정일인 2026년 10월보다 훨씬 전인 7월 9일부터
> 신규 요청에 404를 반환하기 시작한 사례 — Google 개발자 포럼에 보고됨). 그래서 지금은 지정한
> 모델이 404를 반환하면 자동으로 `gemini-flash-latest`(Google이 계속 최신 안정 버전으로
> 유지해주는 별칭)로 한 번 더 시도하도록 만들어뒀어요. 그래도 또 문제가 생기면
> `lib/gemini/analyzer.ts`의 `PRIMARY_MODEL` 상수를
> [Google 공식 모델 목록](https://ai.google.dev/gemini-api/docs/models)에서 "Stable" 표시된
> 모델로 바꿔주세요.

**실제로 Gemini가 쓰이고 있는지 확인하는 방법**
- 채팅에서 카테고리 응답이 오면 말풍선 옆에 "✨ Gemini" 배지가 떠요 (키워드 매칭이면 "🔤 키워드 매칭" 배지)
- 터미널에 `[gemini analyzer] ✅ Gemini 사용됨 — "..." → wrinkle` 같은 로그가 찍혀요
- 실패 시엔 `❌ Gemini 호출 실패` 로그와 함께 자동 폴백돼요 (키 오타, 요청 한도 초과 등을 여기서 확인 가능)

## 실제 서비스 연동 시 다음 단계

| 항목 | 현재 상태 | 연동 방법 |
|---|---|---|
| 자연어 분석 | 키워드 매칭 폴백 | `.env.local`에 `GEMINI_API_KEY` 추가 → `lib/gemini/analyzer.ts`가 자동 전환 |
| 제품 데이터 | `lib/products.ts` 더미 데이터 | `.env.local`에 Supabase 키 추가 → `lib/supabase/client.ts`로 `product_ingredients` 조회, `scoreProducts()`는 그대로 재사용 |

## ⚠️ 안전 관련 — 자극 성분 우선순위 조정

QA 테스트에서 두 가지 문제가 발견됐어요. `lib/safety.ts`의 `applySafetyAdjustment()`가
AI 분류 결과와 상관없이 항상 체크해서 두 경우 모두 대응해요:

1. **이미 자극받은 피부(따가움·붉어짐 등)에 살리실산 같은 각질 제거 성분을 1순위로 추천**
   → 사용자 문장에 자극 신호가 있으면 `Ingredient.irritant: true`로 표시된 성분(살리실산,
   레티놀)을 1순위 추천에서 내리고 진정 성분을 대신 올려요.
2. **레티놀+비타민C처럼 병용 주의가 필요한 조합을 언급했는데 경고만 뜨고 정작 그 성분이
   여전히 "추천" 배지를 달고 있던 문제** → 처음엔 경고 문구만 추가했었는데, 정작 추천
   순위는 안 바뀌어서 모순이었어요. 이제 조합 주의가 감지되면 irritant 성분을 똑같이
   1순위에서 내려요.

두 경우 모두 화면에 별도 경고 박스가 항상 표시돼요. 이 경고는 AI가 생성하는 인사말과 완전히
독립적이에요 — AI 응답이 이상하게 나오는 경우에도 경고만큼은 코드로 직접 체크해서 항상
뜨게 만들었어요. 다른 성분에도 `irritant: true`가 필요하거나, 다른 조합 주의가 필요하면
`lib/safety.ts`의 `KNOWN_CAUTION_PAIRS`에 추가해주세요.

**한계**: 이건 "위험하게 느껴지는 특정 신호가 있으면 자극 성분을 내린다"는 규칙 기반
안전장치예요. QA 리포트에 있던 다른 문제(여러 증상이 섞인 문장에서 맥락 놓침 등)는 지금
구조(AI는 5개 카테고리 중 하나로 분류만 하고, 나머지는 고정 로직)의 근본적인 한계라
이번에도 다루지 못했어요.

## 팀 논의가 필요한 부분 (창래님 피드백)

- **비즈니스 모델**: 지금은 쿠팡 검색 링크로 보내는 게 끝이라 수익 구조가 없어요. 가장
  빠르게 붙일 수 있는 건 **쿠팡 파트너스**(구매 링크에 파트너스 태그만 추가하면 돼요).
  다만 "광고성 리뷰 말고 데이터로 비교하세요"라는 핵심 메시지와 특정 커머스사 종속이
  긴장 관계에 있다는 점은 발표 전에 답을 준비해두면 좋을 것 같아요. 로드맵으로는
  1) 쿠팡 파트너스(즉시) → 2) B2B 트렌드 데이터 판매(6개월 내) → 3) 프리미엄 구독(장기)
  순으로 보여주면 단계적으로 계획된 것처럼 보여요.
- **AI의 실질적 역할**: "그냥 GPT 써도 되지 않나" 질문에는 **"AI 챗봇이 아니라
  전성분 데이터 기반 가성비 계산 엔진 + 자연어 인터페이스"**로 프레이밍하는 걸 추천해요.
  GPT는 실제 전성분 배치 순서를 모르니 환각을 낼 수밖에 없고, 진짜 계산은 여기서
  일어난다는 걸 명확히 하면 돼요. AI 역할은 (1) 카테고리 분류 (2) **추천 이유 생성**
  (`lib/gemini/reasoning.ts` — 이번에 추가했어요, 계산된 점수를 근거로 AI가 문장만
  자연스럽게 다듬어요. 점수 자체는 AI가 계산하지 않아서 환각 위험이 없어요) 두 가지예요.

  히어로와 최종 CTA 두 곳만 "임팩트 포인트"로 남겨뒀어요.
- **민트가 랜딩 밖으로 확장됨**: 채팅 결과 카드(`ResultCard`)의 가성비 지수, 계산법 보기
  모달의 최종 점수, 비교함의 가성비 지수 행, 검색 페이지의 "성분핏 추천템" 뱃지까지
  전부 작은 반짝임 아이콘(`SparkleIcon`) + 민트 텍스트로 통일했어요. "핵심 결과값"에만
  적용했고, 배치 점수·가격 점수 같은 중간 계산값은 그대로 뒀어요 — 최종 숫자만 강조돼야
  의미가 있어서요.
- **민트 컬러 3단계**: `accent`(#5ab6a3, 채도 낮춘 뮤트 톤 — 배경/데이터 바 등에)
  `accent-text`(#1e6c5e, 흰 배경 글자용 — 대비 6.25:1) `accent-deep`(#36786a, 보라→민트
  그라데이션 버튼 전용 — 흰 글자가 그라데이션 양쪽 끝 어디서나 잘 보이게 명도를 낮춘 버전).
  버튼마다 색이 살짝 다르게 느껴진다면 의도한 거예요 — 순수 mint 배경 위 흰 글자는
  대비가 부족해서(2.4:1) 그라데이션 버튼엔 더 진한 버전을 따로 썼어요.
- **버튼 스타일 구분**: 헤더의 "AI 추천받기"는 아웃라인(테두리만), 히어로·최종
  CTA·검색페이지 CTA는 보라→민트 그라데이션(흰 글자)으로 통일했어요. 나머지 버튼(비교함
  담기, 구매하러 가기 등)은 그대로 보라색 단색이에요 — 민트는 "꼭 눌러야 하는" 진입점
  CTA에만 한정했어요.
- **배경 블롭 투명도**: 민트색 블롭만 12%로 확 낮췄어요(보라색은 40%). "은은한 분위기"만
  주는 용도라, 화면 방향에 따라 거의 안 보일 수도 있어요 — 의도한 거예요.
- **폰트**: 제목(h1~h3)은 Gmarket Sans, 본문은 Pretendard로 바꿨어요. 둘 다 CDN(jsDelivr,
  눈누 공식 배포처)에서 불러와서 별도 설치가 필요 없어요. 기존 Geist 폰트는 완전히
  제거했어요(라틴 전용이라 한글엔 어차피 적용 안 되고 있었어요). `h1`, `h2`, `h3` 태그를
  쓰기만 하면 자동으로 Gmarket Sans가 적용되니, 새 페이지 만들 때 폰트를 따로 지정 안
  하셔도 돼요.
- **민트 포인트 컬러 확장**: `--color-accent`(배경·버튼용, 밝은 민트)와
  `--color-accent-text`(글자용, 대비 확보를 위해 더 진하게 조정)로 나눴어요. 흰 배경 위에
  민트를 글자색으로 쓰면 가독성이 떨어질 수 있어서, 텍스트는 항상 `accent-text`를 쓰도록
  구분해뒀어요. "AI 추천 시작/받기" 같은 핵심 CTA 버튼과 가성비 지수 같은 핵심 수치에만
  적용했고, 나머지는 그대로 보라색이에요(대략 8:2 비율 유지).
- **폰트**: 제목(h1~h3)은 Gmarket Sans, 본문은 Pretendard로 바뀌었어요. 둘 다 CDN(눈누
  공식 배포처)에서 불러와요 — `npm install` 같은 별도 설치 없이 그냥 동작해요. 인터넷
  연결이 없으면 시스템 기본 폰트로 자동 대체돼요. 기존 Geist 폰트는 완전히 제거했어요
  (Geist는 라틴 문자 전용이라 한글엔 애초에 적용 안 되고 있었어요).
- **민트 포인트 컬러 확대 적용**: `--color-accent`(배경/버튼용, 밝은 민트)와
  `--color-accent-text`(글자용, 대비를 위해 더 진하게 조정한 민트) 두 가지로 나눴어요.
