## 1. 데이터 확보 전략

**피드백**: 올리브영·쿠팡은 크롤링 약관 위반 소지, 화해는 API 미공개 → 중소기업 포함
특정 회사와의 제휴 고려.

**반영 방법**: 크롤링 배제를 원칙으로 명시하고, 합법 경로를 3단계로 설계했습니다.

1. **공공·공개 데이터 (즉시)** — 식약처 의약품안전나라(기능성화장품 심사·보고 정보),
   대한화장품협회 성분사전, 공공데이터포털.
2. **중소 브랜드 직접 제휴 (핵심)** — "전성분·가격 데이터 제공 ↔ 성분 투명성 배지·추천
   노출" 교환 구조. 후보군: D2C 인디 세럼 브랜드(와디즈 뷰티 펀딩 상위), 자사몰 중심
   중소 책임판매업체, OEM/ODM(한국콜마·코스맥스) 경유 신생 브랜드.
3. **커머스 공식 API (수익 연계)** — 쿠팡 파트너스 오픈 API(약관 위반 없는 공식 경로 +
   제휴 수익), 네이버 쇼핑 검색 API. 보조로 사용자 전성분 사진 제보(검수 후 반영).

**남은 과제**: 제휴 제안서 초안 작성, 1차 컨택 브랜드 리스트업(3~5곳).

## 2. 분석 일관성 검증

**피드백**: temperature 고정, 동일 이미지(입력) 반복 분석 편차 측정 같은 검증 절차 고민.

**반영 방법**:
- **설계 문서화** — 점수를 좌우하는 분류 단계(`lib/gemini/analyzer.ts`)는 이미
  `temperature: 0`으로 결정론적 고정이었음을 명시. 인사말(0.5)·추천 이유(0.4)는 문장
  표현만 담당하고 점수 계산은 AI가 아닌 순수 산식(`lib/scoring/calculator.ts`)이라
  항상 동일하다는 구조적 근거를 함께 서술.
- **검증 절차 구현(MVP)** — `npm run consistency`로 실행하는
  `scripts/consistency-check.mjs` 추가. 카테고리별 명확한 문장 4개 + 모호한 문장 1개를
  각 4회 반복 호출해 문장별 분류 일치율(%)·응답 분포·AI 사용 여부를 표로 출력.
- **통과 기준** — 명확한 문장 일치율 100%, 모호한 문장은 임의 카테고리 강제 분류 대신
  되물음/미지원으로 일관 처리. 키워드 폴백(AI 미사용) 상태의 100%는 검증으로 인정하지
  않도록 경고 표시.

**남은 과제**: 실제 GEMINI_API_KEY 환경에서 측정 결과 수치를 README에 기록, 반복 횟수를
늘린(N=20+) 정밀 측정.

## 4. 전성분 함량순 기재의 법적 한계

**피드백**: 화장품법 시행규칙상 1% 이하 성분·착향제·착색제는 순서 무관 기재 가능한데,
이 서비스의 핵심 성분(레티놀 0.01~0.3%, 아데노신 고시 함량 0.04%, 펩타이드 극미량)이
바로 이 구간 → 한계를 먼저 명시하고, 성분별 기준위치가 이를 보정하는 장치임을 논리적으로
서술할 것.

**반영 방법**: README 가성비 점수 계산 섹션에 "전성분 기재 순서의 법적 한계와 보정 장치"
소단락 추가. 논리 구조:

1. **한계 명시** — 함량순 기재 원칙의 법적 예외(1% 이하·착향제·착색제)와, 핵심 성분이
   그 구간에 있다는 사실을 먼저 인정.
2. **보정 장치** — 절대 순번이 아니라 **성분별 기준위치(refPosition) 대비 상대 배수**로
   평가(`placementScore = 실제위치 ÷ 기준위치` 구간 점수). 기준위치는 각 성분이 유효
   함량일 때 통상 배치되는 위치(레티놀 12, 아데노신 15, 나이아신아마이드 7 등)라, 같은
   순번이라도 성분마다 다르게 평가되어 1% 이하 구간의 왜곡을 완화. 이 로직은 이미 코드로
   구현되어 있었고(문서 서술만 없던 상태), 이번에 논리를 명문화.
3. **잔여 한계 인정** — 기재 순서만으로 정확한 함량 추정은 불가. 향후 기능성 고시 함량
   표기 제품의 공개 정보로 기준위치를 지속 보정하는 것을 다음 단계로 명시.

**남은 과제**: 기능성 고시 함량 공개 데이터(식약처) 수집 → refPosition 값 근거 보강.
